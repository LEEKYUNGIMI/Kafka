< 프로젝트 구상 >
1. 주제 : 이벤트 기반 아키텍처를 적용한 실시간 선착순 쿠폰 발급 시스템
2. 과제 :	1) 수만 명의 사용자가 동시에 몰리는 트래픽 안정적 처리
   2)  데이터베이스 직접적 부하 없이 동시성 문제 해결
   3) 사요자에게 실시간으로 발급 상태 ( 대기, 성공, 실패 ) 피드백

3. 시나리오
    1) 사용자 요청 : 클라이언트에서 "쿠폰 발급" 버튼 클릭  -> API 요청
    2) API 서버 ( 쿠폰 서비스 )
       -> 	- Redis 확인해 이미 발급받은 사용자 체크
       - 유효한 요청 -> Kafka 메세지 전송 ( produce )
       - 사용자에게 즉각적 응답
    3) Kafka (메세지큐) : API서버로부터의 대량 쿠폰 발급 요청 순서대로 큐에 쌓아둠
    4) 쿠폰 컨슈머 (Workwer)
       ->	- Kafka topic 구독하고있다가 새로운 요청 메시지가 들어오면 순차적으로 처리
       - Redis의 INCR명령어 사용해 발급 쿠폰 수 1증가, 현재 발급된 쿠폰 수 총량 초과 학인
       - 성공 : DB에 쿠폰 발급 내역 저장  || 실패 : 별도 처리
       - 처리결과 Kafka topic에 메시지로 전송
    5) 알림
       -> 	- topic 구독하고있다가 처리 결과 메시지 수신
       - WebSocket 통해 사용자에게 실시간으로 발급 결과 전송

< 어필 포인트 >
1. 대용량 트래픽 처리 능력 : Kafka 도입하여 순간적인 트래픽 급증에도 시스템 다운 없이 모든 요청을 안정적으로 큐에 저장, 워커 프로세스가 순차 처리하도록 설계하여 문제 래결
2. 성능 최적화 경험 : 데이터베이스의 직접적인 쓰기 부하 줄이기 위해 Redis의 Atomic 연산을 이용해 실시간 재고를 관리함으로써, DB 병목 현상 방지, 빠른 응답속도 확보
3. 비동기/이벤트 기반 아키텍처 설계 : 서비스간의 의존성 낮추고 확장성 높이기 위해 이벤트 기반 아키텍터 적용. -> 쿠폰 발급 로직과 알림 로직 분리, 유지보수 향상
4. 최신 기술 스택 활용 : Docker, Kafka, Redis 등 현업에서 널리 사용되는 기술 스택 활용, 클라우드 네이티브 환경에 대한 높은 이해도
